# Статус-коды ошибок ATOM

Все коды объявлены в `namespace error_code` в файле
`include/addition/error_manager.hpp`. Любая ошибка, выводимая через
`ErrorManager::add(...)`, либо любое исключение `RuntimeError`/`TypeError`/
`PreparationError`, несёт такой код.

## Формат кода

```
ORIGIN_DDDD
```

- `ORIGIN` — 3-буквенный префикс подсистемы, из которой пришла ошибка:
  - `LEX` — лексер (`src/lexer/lexer.cpp`)
  - `PAR` — парсер (`src/parser/parser.cpp`)
  - `SEM` — семантический анализ (3 прохода: `DefinitionVisitor`, `AnalysisVisitor`, `OptimizationVisitor`)
  - `RUN` — интерпретатор/время выполнения (`src/semantics/value.cpp`,`src/addition/builds_manager.cpp`)
  - `PRE` — подготовка запуска, загрузка файлов (`src/addition/source_manager.cpp`)
  - `CFG` — обработка флагов командной строки (`src/addition/program_manager.cpp`)
- `_` — разделитель.
- `DDDD` — 4 цифры. Первая цифра — категория ошибки внутри подсистемы
  (а для `SEM` — дополнительно кодирует номер прохода, см. ниже), оставшиеся —
  порядковый номер конкретной ошибки внутри этой категории.

Все коды имеют одинаковую длину (8 символов), независимо от подсистемы.

Код ошибки выводится в квадратных скобках в конце каждого диагностического
сообщения компилятора, например:

```
└─ Ожидалось выражение [PAR_2002]
```

## SEM — семантический анализ

Формат `SEM_PTSS`: `P` — номер прохода (`1` — `DefinitionVisitor`/
`SemanticsManager`, `2` — `AnalysisVisitor`, `3` — `OptimizationVisitor`),
`T` — категория внутри прохода, `SS` — порядковый номер.

### Проход 1 (`DefinitionVisitor`, регистрация глобальных сущностей)

Категория 1 — `use`/импорт:

| Код | Описание |
|---|---|
| `SEM_1101` | Встроенный модуль, указанный в `use "@..."`, не найден |
| `SEM_1102` | Обнаружен циклический импорт файла |
| `SEM_1103` | Конфликт имён при полном импорте модуля (`use "..."` без `{ }`) |
| `SEM_1104` | Импортируемый идентификатор не найден в модуле |
| `SEM_1105` | Транзитивный импорт идентификатора запрещён (предупреждение) |
| `SEM_1106` | Конфликт имён при импорте конкретного объекта из модуля |

## CFG — обработка флагов командной строки

`src/addition/program_manager.cpp`. Эти ошибки возникают до запуска
конвейера компиляции (на этапе разбора аргументов `argv`), поэтому
`SourceManager`/`ErrorManager` для них недоступны — код встраивается прямо
в текст сообщения `std::runtime_error` в формате `[CFG_XXXX] текст`.

| Код | Описание |
|---|---|
| `CFG_0001` | После флага `-dir`/`--directory` не указан путь к директории |
| `CFG_0002` | Включены флаги вывода (`-a`/`-t`/`-i`), но не указан `-dir`/`--directory` |
| `CFG_0003` | Не указан файл с исходным кодом (отсутствует обязательный аргумент `argv[1]`) |

## `ERR_0000` — служебный код-заглушка

`error_code::CODE_NULL`. Используется как значение по умолчанию там, где
явный код не передан (в частности — параметр по умолчанию у
`RuntimeError`/`TypeError`, если конструктор вызван без явного кода). В
норме не должен попадать в вывод пользователю: каждое реальное место
выброса ошибки в кодовой базе передаёт собственный явный код.
